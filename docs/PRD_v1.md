## 1. 产品定位与目标

### 产品定位
一款 **AI 驱动的修仙文字冒险游戏**，玩家在仙侠世界中扮演修真者，通过自由文字输入与 AI 叙事引擎互动，体验修炼、战斗、探索、结交 NPC 的完整修仙之旅。移动端开箱即用，无需配置即可游玩。

### 产品目标

| # | 目标 | 衡量指标 |
|---|------|---------|
| G1 | **开箱即玩** | 首次启动 ≤ 3 秒进入游戏，无需任何配置即可开始修仙 |
| G2 | **完整可玩** | 支持从练气到飞升的完整修仙旅程，至少包含 30 分钟完整游戏体验 |
| G3 | **流畅体验** | AI 响应等待时有流式输出和加载动画，用户感知等待 ≤ 2 秒 |

## 2. 用户故事

### US-01 开局体验
**作为**一名新玩家，**我想要**打开 APP 后无需任何配置即可开始修仙，**以便**立即体验游戏乐趣。

### US-02 自由行动
**作为**一名修仙者，**我想要**用自然语言描述我的行动（如"我向长老请教剑法"），**以便**获得沉浸式的修仙体验而非被菜单选项限制。

### US-03 流式叙事
**作为**一名玩家，**我想要**AI 叙事逐字流出而非等待数秒后一次性出现，**以便**在等待时不会焦虑地以为卡死了。

### US-04 境界突破
**作为**一名修仙者，**我想要**在经验积累满后尝试突破境界，并承担失败风险，**以便**体验修仙路上"一步登天"或"走火入魔"的紧张感。

### US-05 战斗系统
**作为**一名修仙者，**我想要**与妖兽或敌对修士进行回合制战斗，**以便**通过策略和实力赢得胜利而非仅靠文字描述。

### US-06 自定义模型
**作为**一名高级玩家，**我想要**在设置中切换到自己的 API 端点（如本地部署的 Ollama 或其他 OpenAI 兼容服务），**以便**降低延迟或使用更强大的模型。

### US-07 存档管理
**作为**一名修仙者，**我想要**拥有多个存档槽位来尝试不同的角色发展路线，**以便**不会因为一次失败就失去所有进度。

### US-08 NPC 交互
**作为**一名修仙者，**我想要**与 NPC 建立关系（拜师、结盟、结仇），**以便**影响世界走向和获取独特资源。

### US-09 新手引导
**作为**一名新玩家，**我想要**在首次游戏时获得简短引导，**以便**了解如何操作和基本游戏规则。

### US-10 游戏结束与重开
**作为**一名修仙者，**我想要**在角色死亡时看到结局画面并可以选择重新开始，**以便**有完整的游戏体验闭环。

## 4. 游戏系统设计

### 4.1 境界体系

#### 完整境界列表（9 大境界）

| 序号 | 境界 | 阶段 | 突破经验需求 | HP 基础 | MP 基础 | 寿命 | 突破成功率 |
|------|------|------|-------------|---------|---------|------|-----------|
| 1 | 练气 | 一至九层 (9阶) | 100/层 | 100 | 50 | 100 | 95% |
| 2 | 筑基 | 初/中/后/圆满 (4阶) | 500/阶 | 300 | 150 | 200 | 80% |
| 3 | 金丹 | 初/中/后/圆满 (4阶) | 2000/阶 | 800 | 400 | 500 | 60% |
| 4 | 元婴 | 初/中/后/圆满 (4阶) | 8000/阶 | 2000 | 1000 | 1000 | 40% |
| 5 | 化神 | 初/中/后/圆满 (4阶) | 30000/阶 | 5000 | 2500 | 3000 | 25% |
| 6 | 合体 | 初/中/后/圆满 (4阶) | 100000/阶 | 12000 | 6000 | 8000 | 15% |
| 7 | 大乘 | 初/中/后/圆满 (4阶) | 300000/阶 | 30000 | 15000 | 20000 | 8% |
| 8 | 渡劫 | 一至九重天劫 (9阶) | 1000000/阶 | 80000 | 40000 | 50000 | 3% |
| 9 | 飞升 | — | — | — | — | 不朽 | — |

#### 突破机制

- **条件**：经验值 ≥ experience_to_next
- **操作**：玩家输入"闭关突破"或类似意图
- **流程**：
  1. World Builder 根据当前境界和灵根等级生成天劫/心魔考验叙事
  2. Judge 评估突破合理性（灵根品级影响成功率修正）
  3. 成功：境界提升，属性大幅增长，解锁新功法/区域
  4. 失败：HP 降至 1，经验值损失 30%，可能获得"走火入魔"状态效果
- **辅助**：丹药/法宝可提升突破成功率 10%-30%
- **实现**：新增 `breakthrough.py` 模块处理突破逻辑和概率计算

### 4.2 战斗系统（回合制）

#### 战斗流程

```
玩家发起战斗 → 进入战斗状态 → 回合循环 → 战斗结束
                                      ↓
                        玩家选择行动 → AI 生成战斗叙事+状态变更
                                      ↓
                        Judge 审核战斗合理性 → 应用 delta
                                      ↓
                        检查 HP → 一方归零则战斗结束
```

#### 战斗状态

```python
class CombatState(TypedDict, total=False):
    in_combat: bool
    opponent_name: str
    opponent_realm: str
    opponent_hp: int
    opponent_hp_max: int
    round_count: int
    combat_log: list[str]
```

#### 战斗操作

| 操作 | 效果 | MP 消耗 |
|------|------|---------|
| 普通攻击 | 基于 realm + weapon 伤害 | 0 |
| 施展功法 | 功法等级 × 属性加成 | 按功法等级 10-50 |
| 使用丹药 | 恢复 HP/MP 或临时增益 | 0 |
| 防御 | 本回合伤害减免 50% | 0 |
| 逃跑 | 概率脱离战斗（低境界对高境界失败率极高） | 0 |

#### 战斗 AI 行为

- Narrator 系统提示词新增战斗模式规则
- 敌方行动由 AI 生成（受 Judge 审核约束）
- 战斗叙事更紧凑（1-2 段/回合）
- delta 中包含双方 HP 变化

## 5. 多模型支持方案

### 5.1 默认 Agens 内置方案

**方案**：在代码中硬编码 Agens 默认 API Key 作为 fallback。

```
配置优先级：用户自定义 API Key > 环境变量 AGNES_API_KEY > 内置默认 Key
```

**实现位置**：`settings.py`

```python
# 内置默认 Key（仅作为 fallback，用户可覆盖）
_BUILTIN_API_KEY = "agn-xxxxxxxxxxxx"  # Agens 官方 Key
_BUILTIN_BASE_URL = "https://apihub.agnes-ai.com/v1"
_BUILTIN_MODEL = "agnes-2.0-flash"

class Settings:
    def __init__(self, **overrides):
        # 优先级：overrides > 用户持久化配置 > 环境变量 > 内置默认
        self.api_key = (
            overrides.get("api_key") or
            self._load_persisted("api_key") or
            os.environ.get("AGNES_API_KEY", "") or
            _BUILTIN_API_KEY
        )
        # ... 同理 base_url, model
```

**安全考虑**：
- 内置 Key 仅在用户未配置时使用
- API Key 在 UI 和日志中始终脱敏显示
- 内置 Key 有速率限制（Agens 侧控制）

### 5.2 自定义模型接入方案

**支持的 OpenAI 兼容格式**：

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| base_url | API 端点 | `https://apihub.agnes-ai.com/v1` |
| model | 模型名称 | `agnes-2.0-flash` |
| api_key | 认证密钥 | 内置 Agens Key |
| temperature | 生成温度 | 0.7 |
| max_tokens | 最大输出 token | 4096 |

**配置持久化**：
- 位置：`runtime/config/user_model.json`
- 格式：
  ```json
  {
    "base_url": "http://localhost:11434/v1",
    "model": "qwen2.5:14b",
    "api_key": "ollama",
    "temperature": 0.8,
    "max_tokens": 4096,
    "is_custom": true
  }
  ```
- 加载时机：APP 启动时读取，运行时修改即时生效

**模型可用性检测**：
- 设置保存时发送测试请求 `/chat/completions`（1 token 测试）
- 失败时提示错误原因，不阻止保存（可能暂时离线）
- 游戏中 API 失败时自动 fallback 到 Agens 默认

```
│  [状态][背包][功法][地图][任务]│
├─────────────────────────────┤
│  [输入框................] [▶] │ ← 底部固定，文字输入 + 发送
└─────────────────────────────┘
```

**设计要点**：
- 叙事区自动滚动到底部
- 新叙事以淡入动画出现
- 流式输出时叙事区实时滚动
- 长按叙事文本可复制
- 竖屏优先，横屏时叙事区更宽

- 用户可跳过（"跳过引导"按钮始终可见）
